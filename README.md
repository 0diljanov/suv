# AquaFlow IoT Platform

Современная веб-панель для управления сетью водоматов — замена устаревшему `iot.sdofjs.com`.

## Что внутри

- **Двуязычный интерфейс** (Русский / O‘zbekcha) с переключателем — выбор сохраняется в браузере.
- **Тёмная тема** в стиле современных IoT-дашбордов (Grafana / Datadog).
- **4 роли пользователей**, каждая видит свой набор разделов:
  - **Администратор** — полный доступ, включая управление пользователями.
  - **Менеджер** — водоматы, финансы, отчёты, заявки.
  - **Техник** — водоматы, карта, уведомления, заявки.
  - **Владелец / Инвестор** — дашборд, финансы, отчёты.
- **Push-уведомления о неполадках** (toast) — затор купюроприёмника, фильтр требует замены, потеря связи, низкий запас воды, инкассация и т.д.
- **Геокарта водоматов** на Leaflet — статусные маркеры и попапы с метриками.
- **Графики** на Chart.js — динамика выручки, доли способов оплаты.
- **Карточка устройства с диагностикой узлов**: купюроприёмник, монетоприёмник, помпа, фильтр, УФ-лампа, QR-сканер, картридер, дверь.
- **Привязка реальных устройств**: форма «Привязать водомат» сохраняет серийники в localStorage. По умолчанию подключён реальный контроллер LFT-780 / Air780E с серийником 10992.

## Деплой на GitHub Pages

В этом репозитории уже настроен workflow `.github/workflows/deploy.yml` — при каждом push в ветку `main` сайт автоматически публикуется на GitHub Pages.

### Первичная настройка (один раз)

1. Создайте репозиторий `Suv` на GitHub (Public).
2. Запушьте код:
   ```bash
   git remote add origin https://github.com/<USERNAME>/Suv.git
   git push -u origin main
   ```
3. Откройте `Settings → Pages` репозитория.
4. В разделе **Build and deployment → Source** выберите **`GitHub Actions`**.
5. Перейдите в `Actions` — workflow «Deploy to GitHub Pages» уже запустился и опубликует сайт через ~1 минуту.

Дальше любой `git push origin main` будет автоматически обновлять сайт.

## Локальный запуск

Просто откройте `index.html` в браузере — для демо-просмотра дополнительных шагов не требуется.

Или запустите простой HTTP-сервер:

```bash
cd Suv
python3 -m http.server 8000
# открыть http://localhost:8000/
```

## Что улучшено относительно iot.sdofjs.com

| Оригинал | AquaFlow |
|---|---|
| Только мобильная вёрстка, на десктопе форма скрыта | Адаптивная: десктоп → планшет → телефон |
| Декоративный фон (волны, шарики, иероглифы) | Чистая тёмная тема, нейтральная типографика |
| Обрезанный текст: «Pay An-», «Remot Boot», «Consum Recharge» | Полные термины с переводом на RU/UZ |
| Алерты — только счётчики цифр | Лента уведомлений + push-toast + фильтры по тяжести |
| Нет геокарты | Карта с маркерами по статусу |
| Нет графиков, только сырые числа | Динамика выручки + структура оплат |
| Нет страницы устройства с диагностикой | Полная карточка: давление, температура, сигнал, узлы |
| Нет ролей / разграничения | 4 роли с раздельной навигацией |
| Только английский с сокращениями | RU + UZ, понятные термины |
| Безопасность: «123456» | Демо-вход без проверки + переход на бэк по готовности |

## Подключение реального бэкенда

Сейчас данные генерируются в `assets/js/data.js` (объект `DATA`). Чтобы подключить ваш backend:

1. Замените массивы на `fetch('/api/...')`.
2. При логине отправляйте `POST /auth/login` → токен в `localStorage` под ключом `aquaflow.session`.
3. WebSocket для realtime — заменить `startRealtime()` в `app.js`.
4. На стороне контроллера (Air780E) — пушить телеметрию по серийнику (например, 10992) на ваш сервер.
